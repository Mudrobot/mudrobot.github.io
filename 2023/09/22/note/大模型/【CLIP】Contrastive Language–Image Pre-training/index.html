Contrastive Language–Image Pre-training

文章内容

Abstract

State-of-the-art computer vision systems are trained to predict a ﬁxed set of predetermined object categories. This restricted form of supervision limits their generality and usability since additional labeled data is needed to specify any other visual concept. Learning directly from raw text about images is a promising alternative which leverages a much broader source of supervision. We demonstrate that the simple pre-training task of predicting which caption goes with which image is an efﬁcient and scalable way to learn SOTA image representations from scratch on a dataset of 400 million (image, text) pairs collected from the internet. After pre-training, natural language is used to reference learned visual concepts (or describe new ones) enabling zero-shot transfer of the model to downstream tasks. We study the performance of this approach by benchmarking on over 30 different existing computer vision datasets, spanning tasks such as OCR, action recognition in videos, geo-localization, and many types of ﬁne-grained object classiﬁcation. The model transfers non-trivially to most tasks and is often competitive with a fully supervised baseline without the need for any dataset speciﬁc training. For instance, we match the accuracy of the original ResNet-50 on ImageNet zero-shot without needing to use any of the 1.28 million training examples it was trained on. We release our code and pre-trained model weights at https://github.com/OpenAI/CLIP.

最先进的计算机视觉系统经过训练可以预测一组固定的预定对象类别。这种受限的监督形式限制了它们的通用性和可用性，因为需要额外的标记数据来指定任何其他视觉概念。直接从原始文本中学习图像是一种很有前途的替代方案，它利用了更广泛的监督来源。我们证明，预测哪个标题与哪个图像对应的简单预训练任务是一种高效且可扩展的方法，可以在从互联网收集的 4 亿对（图像、文本） 数据集上从头开始学习 SOTA 图像表示。预训练后，使用自然语言来引用学习的视觉概念（或描述新的视觉概念），从而实现模型零样本传输到下游任务。我们通过对 30 多个不同的现有计算机视觉数据集进行基准测试来研究这种方法的性能，涵盖 OCR、视频中的动作识别、地理定位和多种类型的细粒度对象分类等任务。该模型可以轻松地迁移到大多数任务，并且通常可以与完全监督的基线竞争，而无需任何数据集特定的训练。例如，我们在 ImageNet 零样本上匹配原始 ResNet-50 的准确性，而无需使用其所训练的 128 万个训练样本中的任何一个。我们在 https://github.com/OpenAI/CLIP 发布了代码和预训练模型权重。

总结

CLIP出现的目的

尽管深度学习已经彻底改变了计算机视觉，但目前的方法存在几个主要问题：

  1. 典型的视觉数据集是劳动密集型的，创建成本很高，而只教授一组狭窄的视觉概念;
  2. 标准视觉模型擅长一项任务和一项任务，并且需要付出巨大的努力来适应新任务

为了解决上面的这些问题，我们提出CLIP，他使用互联网上大量自然语言监督对各种图像进行训练，可以用自然语言指示网络执行各种分类基准测试。因为不直接针对基准测试的性能进行优化，使得这个模型更加具有代表性。

CLIP最终在ImageNet上取得了比ResNet101更好的性能。

CLIP 实验结果展示

核心思想

扩展一个简单的预训练任务足以在各种图像分类数据集上实现有竞争力的零镜头性能。我们的方法使用大量可用的监督来源：文本与互联网上找到的图像配对。此数据用于为 CLIP 创建以下代理训练任务：给定一张图像，预测一组 32,768 个随机采样文本片段中的哪一个实际上与我们的数据集中的它配对。

目标函数构造的是一个对称的contrastive loss，包含image-to-text 和 text-to-image两个损失函数。如下所示

image-to-text 对比损失计算:

Li2t(i)=−log⁡exp⁡(s(Vi,Ti))∑a=1Bexp⁡(s(Vi,Ta))L_{i2t}(i) = -\log\frac{\exp(s(V_i,T_i))}{\sum_{a=1}^B\exp(s(V_i,T_a))} Li2t​(i)=−log∑a=1B​exp(s(Vi​,Ta​))exp(s(Vi​,Ti​))​

text-to-image 对比损失计算:

Lt2i(i)=−log⁡exp⁡(s(Vi,Ti))∑a=1Bexp⁡(s(Vi,Ta))L_{t2i}(i) =-\log\frac{\exp(s(V_i,T_i))}{\sum_{a=1}^B\exp(s(V_i,T_a))} Lt2i​(i)=−log∑a=1B​exp(s(Vi​,Ta​))exp(s(Vi​,Ti​))​

总的Loss函数就是：上面两个loss相加。

Zero-shot 过程

  • Sample所有N个class，得到N个input text，都经过text encoder编码得到对应的N个class text embedding（我这里之所叫embedding而不叫representation是想说明这个特征是经过encoding和projection得到的）；
  • Sample一个要预测的image，得到其image embedding；
  • 以N个text embedding为key，以当前image embedding为query，算cosine相似度，相似度最高的即为Top-1的prediction class。

算法评价

解决的问题（优点）

  1. 解决了数据集标注昂贵的问题，CLIP直接从互联网中公开的文本图像对进行学习。
  2. 学习了更广阔的视觉概念，学习的视觉概念不局限于数据集中的视觉概念了
  3. 解决了之前计算机视觉方法，实际性能和理论性能之间的差距问题。（因为训练只是使用数据集，导致当域发生变化时，实际性能下降）

局限性

  1. 对于具体任务来说效果还是不错，但是对于抽象任务来说（比如统计图中物体数量，或者预测图片中汽车距离xx建筑的距离等）效果就不够理想，在以上两个例子中，CLIP所给出的结果仅比随机猜测略好。细粒度分类CLIP也存在一定的困难，比如区分汽车模型，飞机变体或花种。
  2. CLIP Zero-shot有时对文本措辞比较敏感，可能需要适当修改文本表述，才能让CLIP取得更好的效果。
