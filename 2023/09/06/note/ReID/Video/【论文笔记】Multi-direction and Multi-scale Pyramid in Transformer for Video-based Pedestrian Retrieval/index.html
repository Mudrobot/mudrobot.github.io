    文章内容

    Abstract

    在视频监控中，行人检索（也称为行人重新识别）是一项关键任务。该任务旨在从不重叠的摄像机中检索感兴趣的行人。最近，基于Transformer的模型在这项任务上取得了重大进展。然而，这些模型仍然存在忽略细粒度、部分知情信息的问题。本文提出了一种多方向、多尺度的 Pyramid in Transformer (PiT) 来解决这个问题。在基于 Transformer 的架构中，每个行人图像被分成许多块（patches）。然后，这些patches被馈送到 transformer layers 以获得该图像的特征表示。为了探索细粒度的信息，本文提出对这些patches应用垂直分割和水平分割来生成不同方向的人体部位。这些部分提供了更细粒度的信息。为了融合多尺度特征表示，本文提出了一种包含全局级信息和来自不同尺度的许多局部级信息的金字塔结构。来自同一视频的所有行人图像的特征金字塔被融合以形成最终的多方向和多尺度特征表示。两个具有挑战性的基于视频的基准 MARS 和 iLIDSVID 的实验结果表明，所提出的 PiT 实现了最先进的性能。广泛的消融研究证明了所提出的金字塔结构的优越性。代码开源在 https://git.openi.org.cn/zangxh/PiT.git.

    In video surveillance, pedestrian retrieval (also called person re-identification) is a critical task. This task aims to retrieve the pedestrian of interest from non-overlapping cameras. Recently, transformer-based models have achieved significant progress for this task. However, these models still suffer from ignoring fine-grained, part-informed information. This paper proposes a multi-direction and multi-scale Pyramid in Transformer (PiT) to solve this problem. In transformer-based architecture, each pedestrian image is split into many patches. Then, these patches are fed to transformer layers to obtain the feature representation of this image. To explore the fine-grained information, this paper proposes to apply vertical division and horizontal division on these patches to generate different-direction human parts. These parts provide more fine-grained information. To fuse multi-scale feature representation, this paper presents a pyramid structure containing global-level information and many pieces of local-level information from different scales. The feature pyramids of all the pedestrian images from the same video are fused to form the final multi-direction and multi-scale feature representation. Experimental results on two challenging video-based benchmarks, MARS and iLIDS-VID, show the proposed PiT achieves state-of-the-art performance. Extensive ablation studies demonstrate the superiority of the proposed pyramid structure. The code is available at https://git.openi.org.cn/zangxh/PiT.git.

    1. Introduction

    图1：Transformer 中的多方向和多尺度金字塔，用于基于视频的行人检索。不同层采用不同方向的划分策略。经过变压器层的处理后，提取部分信息特征。每层包含不同尺度的特征。4个层分别含有1,Dv,Dh,Dp1,D_v,D_h,D_p1,Dv​,Dh​,Dp​的特征

    行人检索是智能监控中的一项关键任务[1] [2] [3]。给定行人图像作为查询，行人检索旨在在大型图库中找到正确的图像。查询图像和匹配的图库图像必须来自不同的相机。行人检索在社会和工业界都有重要的实际应用，例如寻找犯罪嫌疑人、跟踪行人运动等。与基于图像的行人检索相比，基于视频的行人检索可以提供更多的行人步态和视图信息，并减轻遮挡情况的负面影响。因此，基于视频的行人检索越来越受到研究人员的关注[4]。

    对于基于 CNN 的行人检索，将特征图划分为多个水平条纹并单独训练每个水平条纹是通用操作。训练结束后，所有条纹特征被组合起来生成每个图像的卷积描述符，这为模型提供了丰富的特征表示[5]。基于水平划分策略，提出了条纹金字塔来利用每个行人的部分信息[6]。上述这些方法虽然提高了模型性能，但划分策略的方向有限。

    最近，Transformer结构在计算机视觉方面取得了令人难以置信的进步。 Transformer结构是自然语言处理（NLP）中流行的模型，可以有效地处理序列数据。在计算机视觉中，输入图像被分成许多块（patches）。这些patches被视为与 NLP 任务中的单词相似的标记。这些patches的一系列特征嵌入被馈送到Transformer Layers。借助多头自注意力模块，Transformer可以在不丢失信息的情况下获得所有patches之间的全局级关系。同时，CNN卷积核只能感知有限的范围，下采样操作不可避免地会丢失大量信息。此外，与CNN[7]相比，Transformer结构取得了有竞争力的性能。虽然Transformer具有全局感知，但细粒度信息可能被忽略，这导致性能有限。

    本文提出了一种用于视频行人检索的多方向、多尺度的 Transformer 金字塔（PiT），如图 1 所示。该金字塔包含四层，采用“不划分”、垂直、水平和基于patches的方式。分别是划分策略。第一层包括行人图像的全局特征。第二、第三和第四层包含 Dv、Dh 和 Dp 部分级特征。这样，所提出的 PiT 在行人图像中应用多方向分割策略，并为每个行人图像提取多尺度特征表示。

    具体来说，每个行人图像被分成许多块。类标记和所有补丁的特征嵌入被展平并馈送到多个transformer layers。然后，将处理后的patches的token根据其原始位置重新排列成二维结构。这种二维结构采用不同的划分策略，生成不同方向的parts。同一部分内的class token和patch tokens被展平以形成新的token序列。经过最后一个transformer layer的处理后，class token学习多方向的部分信息。通过组合来自不同层的不同尺度的所有特征，获得每个行人图像的特征金字塔。 同一视频中所有图像的对应特征被融合，生成最终的多方向、多尺度特征金字塔。

    传统的基于CNN的方法通常对特征图进行水平划分[5]，这是合理的，因为每个水平条纹通常包含头部、躯干或腿部。然而，垂直划分可以将人体分为右肢、头部和躯干，以及左肢，引入了更多维度的部分知情线索。同时应用垂直和水平划分，形成基于块的划分，也可以提供更细粒度的信息。结合这些多方向、多尺度特征可以有效提高模型性能。在两个具有挑战性的基于视频的基准测试 MARS [8] 和 iLIDS-VID [9] 上进行的实验表明，所提出的 PiT 实现了最先进的性能。广泛的消融研究也证明了所提出的金字塔结构的优越性。

    本文的主要贡献可概括如下：

    • 多方向：变压器中提出的垂直和水平划分策略引入了来自不同方向的细粒度、部分信息信息。
    • 多尺度：不同尺度的全局和局部特征形成特征金字塔。这种多尺度组合使得特征表示丰富且具有辨别力。
    • 性能：所提出的 PiT 在两个具有挑战性的基于视频的基准上实现了最先进的性能，并且广泛的消融研究证明了所提出的多方向和多尺度金字塔结构的优越性。

    Pedestrian retrieval is a critical task in intelligent surveillance[1] [2] [3]. Given a pedestrian image as the query, pedestrian retrieval aims to find the right images in a large gallery. The query image and the matched gallery images must be from different cameras. Pedestrian retrieval has important practical applications in both society and industry, such as finding the criminal suspects and tracking pedestrian movement. Compared to the image-based pedestrian retrieval, the video-based one can provide much more gait and view information of pedestrians and alleviate the negative effects of occlusion situations. Therefore, video-based pedestrian retrieval is getting more and more attention from researchers [4].

    For CNN-based pedestrian retrieval, dividing the feature map into multiple horizontal stripes and individually training each one are general operations. After the training is complete, all the stripe features are assembled to generate the convolution descriptor for each image, which provides the model a rich feature representation [5]. Based on the horizontal division strategy, a pyramid of stripes is proposed to exploit the partial information of each pedestrian [6]. Although these methods above improve the model performance, the direction of division strategy is limited.

    Recently, the transformer structure has achieved incredible progress in computer vision. The transformer structure is a popular model in natural language processing (NLP) and can handle the sequence data effectively. In computer vision, the input image is split into many patches. These patches are regarded as tokens similar to the words in the NLP task. A sequence of feature embedding of these patches is fed to the transformer layers. With the help of the multi-head self-attention module, the transformer can obtain global-level relationships among all the patches without losing information. Meanwhile, the CNN convolution kernel can only perceive limited scope, and the down-sampling operation inevitably loses much information. Moreover, the transformer structure has achieved competitive performance compared with CNN [7]. Although the transformer has a global perception, the fine-grained information may be neglected, which results in a limited performance.

    This paper proposes a multi-direction and multi-scale Pyramid in Transformer (PiT) for video pedestrian retrieval, as illustrated in Fig. 1. The pyramid contains four layers, which adopt “no division”, vertical, horizontal, and patch-based division strategies, respectively. The first layer includes a globallevel feature of the pedestrian image. The second, third, and fourth layers contain Dv, Dh, and Dp part-level features. In this way, the proposed PiT applies multi-direction division strategies in the pedestrian image and extracts a multi-scale feature representation for each pedestrian image.

    Concretely, each pedestrian image is split into many patches. A class token and the feature embeddings of all patches are flattened and fed to multiple transformer layers. Then the processed patch tokens are rearranged into a twodimension structure according to their original positions. Different division strategies are applied to this two-dimension structure, which generates different-direction parts. The class token and patch tokens within the same part are flattened to form a new token sequence. After the process of the last transformer layer, the class token learns the multi-direction partinformed information. A feature pyramid for each pedestrian image is obtained by combining all the features with different scales from different layers. The corresponding features of all the images within the same video are fused to generate the final multi-direction and multi-scale feature pyramid.

    The traditional CNN-based methods usually apply horizontal division to feature map [5], which is reasonable because each horizontal stripe usually contains the head, torso, or legs. However, the vertical division can divide the human body into the right limb, head and torso, and the left limb, which introduces part-informed clues with more dimensions. Applying vertical and horizontal division simultaneously, which forms the patch-based division, can also provide more finegrained information. Combining these multi-direction and multi-scale features can effectively improve the model performance. Experiments on two challenging video-based benchmarks, MARS [8] and iLIDS-VID [9], show the proposed PiT achieves state-of-the-art performance. Extensive ablation studies also demonstrate the superiority of the proposed pyramid structure.

    The main contributions of this paper can be summarized as follows:

    • Multi-direction: the proposed vertical and horizontal division strategies in transformer introduce fine-grained, part-informed information from different directions.
    • Multi-scale: the global and local-level features with different scales form a feature pyramid. This multi-scale combination makes the feature representation rich and discriminative.
    • Performance: the proposed PiT achieves state-of-theart performance on two challenging video-based benchmarks, and extensive ablation studies demonstrate the superiority of the proposed multi-direction and multiscale pyramid structure.

    2. Related Work

    A. Video-based Pedestrian Retrieval

    早期基于图像的行人检索方法融合了RGB、HSV、HoG、LOMO等多种特征，以实现多特征融合并克服行人外观变化的挑战[10]。而基于视频的行人检索对于每个行人都有多个图像。这些连续的行人图像可以提供丰富的时空信息，可以减轻外观变化、遮挡、姿势变化等的负面影响[11]。因此，现有方法侧重于利用行人视频中的空间和时间线索。 GRL[12]采用视频级特征来指导相关图的生成，并将帧级特征分解为高相关性和低相关性特征。 BiCnet-TKS [13]引入了双边互补网络来挖掘每个行人不同的身体部位，并提出了一个时间核选择模块来自适应地探索时间关系。 CTL[14]采用关键点估计器来提取多尺度语义特征以形成拓扑图。然后使用 3D 图卷积来捕获层次空间和时间依赖性。此外，AGW+[15]采用帧级平均池化进行视频特征表示，简单而有效。

    An early method for image-based pedestrian retrieval fused various features, such as RGB, HSV, HoG, LOMO, etc., to achieve the multi-feature fusion and overcome the challenge from pedestrian appearance changes [10]. Whereas videobased pedestrian retrieval has multiple images for each pedestrian. These consecutive pedestrian images can provide abundant temporal and spatial information, which can alleviate the negative effects of appearance change, occlusion, pose variation, etc [11]. Therefore, existing methods focus on exploiting both spatial and temporal clues from pedestrian video. GRL [12] employ video-level features to guide the generation of correlation map and disentangle the frame-level features into high-correlation and low-correlation features. BiCnet-TKS [13] introduced a bilateral complementary network to mine the divergent body parts of each pedestrian and proposed a temporal kernel selection module to explore temporal relations adaptively. CTL [14] employed a key-point estimator to extract multi-scale semantic features to form a topology graph. Then a 3D graph convolution is used to capture hierarchical spatial and temporal dependencies. Besides, AGW+ [15] employed a frame-level average pooling for video feature representation, which is simple but effective.

    B. Vision Transformer

    最近，Transformer 架构已成为自然语言处理事实上的标准。 ViT [7] 将这种架构引入计算机视觉，并在图像分类任务上取得了比许多最先进的方法更好的性能。在这一改进之后，提出了许多工作来提高基于 Transformer 的框架的性能。对于基于视频的分类任务，Swin Transformer [16]提出了一种分层结构，并利用移位窗口来解决非重叠块划分问题。还提出了 3D 移位窗口来预处理视频数据。

    上述这些方法证明了 Transformer 结构可以很好地执行视频分类任务。然而，基于视频的行人检索与视频分类任务有很大不同。基于视频的行人检索任务高度依赖于外观信息而不是运动信息。因此，基于视频的行人检索任务需要能够感知更细粒度信息的 Transformer 结构。

    Recently, transformer architecture has become a de-facto standard for natural language processing. ViT [7] introduced this architecture to computer vision and achieved better performance than many state-of-the-art methods on the image classification task. Following this improvement, many works were proposed to improve the performance of the transformerbased framework. For the video-based classification task, Swin transformer [16] proposed a hierarchical structure and utilized shifted windows to solve the non-overlapping patch division problem. A 3D shifted window is also proposed to preprocess video data.

    These methods above demonstrated the transformer structure could perform well for the video classification task. However, the video-based pedestrian retrieval is very different from the video classification task. The video-based pedestrian retrieval task depends highly on appearance information rather than motion information. Therefore, the transformer structure with the ability to perceive more fine-grained information is needed for the video-based pedestrian retrieval task.

    C. Division Strategies for Pedestrian Retrieval

    对于行人检索任务，将特征图划分为多个条带是典型的操作。 PCB[5]提出将特征图水平划分为多个条带，与原始模型相比取得了显着的性能提升。SPP[17]将特征图分成相等的块，并将多尺度特征块组装成金字塔结构。 HPP[6]采用水平划分形成多尺度金字塔，并提高了该任务的模型性能。

    尽管上述模型采用了不同的划分策略，但尚未在统一框架下进行不同划分策略之间的性能比较，因此这些策略之间的差异仍有待探讨。

    For the pedestrian retrieval task, dividing the feature map into multiple stripes is a typical operation. PCB [5] proposed to horizontally divide the feature map into multiple stripes and achieved significant performance improvement compared with the original model. SPP [17] divided the feature map into equal patches and assembled the multi-scale feature patches into a pyramid structure. HPP [6] employed the horizontal division to form a multi-scale pyramid and improved the model performance for this task.

    Although the models above utilized different division strategies, performance comparison among different division strategies in a unified framework has not been conducted, thus the differences between these strategies remain to be explored.

    The Vision Transformer (ViT) [7] is employed to construct the framework, as illustrated in Fig. 2. Given some pedestrian videos {V1,V2,⋯ }\{V_1,V_2,\cdots\}{V1​,V2​,⋯} and pedestrian IDs {y1,y2,⋯ }\{y_1,y_2,\cdots\}{y1​,y2​,⋯}, each video VVV contains KKK pedestrian images III, as V={I1,I2,⋯ ,IK}V=\{I_1,I_2,\cdots,I_K\}V={I1​,I2​,⋯,IK​}. A convolution layer is used to embed the pedestrian image into multiple feature embeddings. Concretely, after applying convolution operation on the pedestrian image, a feature map f∈Rh×ω×cf\in \mathbb{R}^{h\times\omega\times c}f∈Rh×ω×c is obtained. Then the feature map is flattened to generate NNN features, where N=h⋅ωN = h · \omegaN=h⋅ω and the size of each feature is 1×c1 \times c1×c. In this way, each feature can be treated as the feature embedding of each image patch, and the size of each image patch is the same as convolution kernel size kkk. The convolution stride sss determines the interval of the adjacent image patches.

    The feature embedding of each image patch from the image III is also called patch token ppp. A class token ϕcls\phi_{cls}ϕcls​ with the size of 1×c1 \times c1×c is also introduced to represent the feature embedding of the whole image. After flattening class and patch tokens into a sequence {ϕcls;p1;⋯ ;pN}\{\phi_{cls}; p_1; \cdots ; p_N \}{ϕcls​;p1​;⋯;pN​}, the patch tokens lose the location information in original pedestrian image. Therefore, a position embedding ϕpos∈R(N+1)×c\phi_{pos} \in \mathbb{R}^{(N+1)\times c}ϕpos​∈R(N+1)×c is employed to retain this location information.     总结

    核心思想

    本文旨在提出一种模型，解决之前模型中所存在的忽略 fine-grained part-informed 信息的问题。主要是提出了一种多方向，多尺度的金字塔形的基于Transformer的模型来解决这个问题

    实现过程

    1. 对于每个视频的每个帧，我们将其分成许多块，使用CNN进行特征提取
    2. 类标记和每个块的embedding被展平并送入多个Transformer layer
    3. 然后我们根据这些特征的顺序将他们排列为二维结构
    4. 对重新排列的特征，采用不同的划分方式，划分后展平送入Transformer layer中。
    5. 最后进行特征融合。得到每个帧的特征金字塔
    6. 同一视频中所有图像的对应特征被融合，生成最终的多方向、多尺度特征金字塔
